## UART TX DMA with Interrupt - Basic Concept

In the previous UART TX Polling project, the CPU was responsible for transmitting every character.

The flow was:

```text
CPU
↓
Check TXE flag
↓
Write one byte to USART_DR
↓
Check TXE flag again
↓
Write next byte
↓
Continue until entire message is transmitted
```

This technique is called polling because the CPU continuously checks the TXE flag and waits until the Data Register becomes empty.

The disadvantage is that the CPU is involved in every transmission.

To solve this problem, we use DMA.

```text
DMA
=
Direct Memory Access
```

DMA is a hardware peripheral whose job is to transfer data between memory and peripherals without continuous CPU involvement.

Instead of the CPU moving every byte, we configure DMA once and DMA performs the transfer automatically.

The flow becomes:

```text
CPU configures DMA
↓
CPU starts DMA transfer
↓
CPU free to do other work
↓
DMA transfers data automatically
```

However, after the DMA transfer is completed, we still need a way to know that the transfer has finished.

For this purpose, we use an interrupt.

Instead of continuously checking whether DMA has completed the transfer, we tell the DMA controller:

```text
When transfer is completed,
generate an interrupt.
```

The flow becomes:

```text
CPU configures DMA
↓
CPU starts DMA
↓
CPU free to do other work
↓
DMA transfers data
↓
Transfer Complete Flag Set
↓
DMA Interrupt Generated
↓
Interrupt Service Routine Executes
```

Therefore, in this project:

```text
DMA
↓
Performs data transfer automatically
```

```text
Interrupt
↓
Notifies CPU when transfer is completed
```

---

## Technical Explanation

In this project, DMA is configured in:

```text
Memory-to-Peripheral Mode
```

because:

```text
Source
=
Memory Buffer

Destination
=
USART2 Data Register (USART_DR)
```

DMA automatically:

```text
Read byte from memory
↓
Write byte to USART_DR
↓
Read next byte
↓
Write next byte to USART_DR
```

until all bytes are transferred.

When the final byte is transferred:

```text
Transfer Complete Flag (TCIF)
↓
Set by Hardware
↓
DMA Interrupt Generated
↓
ISR Executes
```

Inside the ISR, we can perform any required action such as:

- Turn ON LED
- Clear Interrupt Flag
- Indicate transfer completion

Therefore, the CPU is only responsible for:

- Configuring DMA
- Providing source address
- Providing destination address
- Providing transfer length
- Starting DMA
- Handling transfer complete interrupt

The actual movement of data is completely handled by DMA.

---

## One-Line Summary

In UART TX DMA with Interrupt, DMA automatically transfers data from memory to the USART2 Data Register, and an interrupt is generated when the transfer is completed, allowing the CPU to perform other tasks during the transfer.

## DMA Stream and Channel Selection

Now let's come to the DMA initialization function.

Before going into the code, we have to remember two important things.

### i.

First, we are going to work with the DMA1 module.

DMA1 is connected to the AHB1 bus, therefore later we will have to enable clock access to DMA1 before using any DMA1 registers.

### ii.

The second important thing is that after deciding to use DMA1, we must determine which Stream and Channel should be used.

For this purpose, we refer to the DMA request mapping table shown in the reference manual.

Since our project uses:

```text
USART2_TX
```

we look for USART2_TX in the table.

From the table we can see:

```text
USART2_TX
↓
DMA1
↓
Stream 6
↓
Channel 4
```

Therefore, for UART TX DMA:

```text
DMA Module  = DMA1
Stream      = Stream 6
Channel     = Channel 4
```

These values are fixed according to the DMA request mapping table provided by ST.

The reason we select Stream 6 and Channel 4 is because the USART2 transmitter is internally connected to this DMA request line.

Therefore whenever USART2 requests a DMA transfer:

```text
USART2_TX
↓
DMA Request
↓
DMA1 Stream 6
↓
Channel 4
↓
Transfer Data
```

All DMA configurations performed later in the code will therefore be done on:

```text
DMA1_Stream6
```

and Channel 4 will be selected inside the stream configuration register.

## DMA initialisation function

Now let us come to the DMA initialization function.

There are a list of configurations that must be performed before DMA can transfer data from memory to USART2.

### 1. Enable clock access to DMA1

Since DMA1 is connected to the AHB1 bus, we must first enable clock access to DMA1 before accessing any DMA registers.

### 2. Disable DMA1 Stream6

Before configuring a DMA stream, the stream must be disabled.

DMA registers should not be modified while the stream is active.

Therefore, we first disable Stream6.

### 3. Wait until DMA1 Stream6 is completely disabled

After clearing the enable bit, the hardware may require a few clock cycles to fully disable the stream.

Therefore, we continuously check the enable bit and wait until the hardware confirms that Stream6 has been disabled.

Only after this step is completed can the DMA stream be safely configured.

### 4. Clear all interrupt flags of Stream6

DMA may contain old interrupt flags from a previous transfer.

These flags could indicate:

- Transfer Complete
- Half Transfer
- Transfer Error
- FIFO Error
- Direct Mode Error

Therefore, before starting a new transfer, we clear all interrupt flags associated with Stream6.

This ensures that the new DMA transfer starts with a clean state.

### 5. Set the destination buffer

Next, we configure the destination address.

Since this project uses Memory-to-Peripheral transfer mode, the destination is the USART2 Data Register.

DMA will eventually copy data to this destination location.

### 6. Set the source buffer

Next, we configure the source address.

The source is the memory buffer that contains the message to be transmitted.

DMA will read data from this memory location.

### 7. Set the transfer length

DMA must know how many data items need to be transferred.

Therefore, we configure the transfer length.

This tells DMA how many bytes must be moved from memory to USART2.

### 8. Select Stream6 Channel4

As discussed previously using the DMA request mapping table:

```text
USART2_TX
↓
DMA1 Stream6
↓
Channel 4
```

Therefore Channel 4 must be selected for Stream6.

This creates the connection between USART2_TX and DMA1 Stream6.

### 9. Enable Memory Increment Mode

During transmission, DMA must read consecutive bytes from memory.

For example:

```text
H
e
l
l
o
```

Each character is stored at a different memory location.

Therefore after reading one byte, DMA should automatically move to the next memory address.

This is achieved using Memory Increment Mode.

### 10. Configure Transfer Direction

In this project, data moves:

```text
Memory
↓
USART2 Peripheral
```

Therefore the transfer direction is configured as:

```text
Memory-to-Peripheral
```

### 11. Enable DMA Transfer Complete Interrupt

After DMA finishes transferring all bytes, we want the CPU to be notified.

Therefore we enable the Transfer Complete Interrupt.

When the final byte is transferred:

```text
Transfer Complete Flag Set
↓
DMA Interrupt Generated
```

### 12. Enable Direct Mode and Disable FIFO

DMA supports FIFO operation as well as Direct Mode.

In this project, Direct Mode is used.

Therefore data is transferred directly between memory and the peripheral without using the FIFO buffer.

### 13. Enable DMA1 Stream6

After all configurations are completed, Stream6 is enabled.

Once enabled, DMA is ready to perform transfers whenever a DMA request is generated.

### 14. Enable USART2 DMA Transmission

USART2 must be informed that DMA will be used for transmission.

After enabling this feature:

```text
USART2
↓
Generates DMA requests
↓
DMA performs transfer
```

### 15. Enable DMA Interrupt in NVIC

Finally, the DMA1 Stream6 interrupt is enabled in the NVIC.

This allows the Cortex-M4 processor to receive DMA interrupts.

Therefore when the transfer completes:

```text
DMA Transfer Complete
↓
DMA Interrupt Generated
↓
NVIC Receives Interrupt
↓
CPU Jumps to ISR
```

At this point, the DMA configuration is complete and DMA is ready to transfer data from memory to USART2 automatically.

### 4. Clear all interrupt flags of Stream6

#### Code


```text
Transfer Error Flag
```

Writing 1 clears the corresponding interrupt flag.

This ensures old DMA events do not affect the new transfer.

## Main Function

### i.

First we create a character array:

```c
char message[31] = "Hello from Stm32 DMA transfer\n\r";
```

This message is stored in SRAM and will act as the source buffer for the DMA transfer.

### ii.

Next we enable clock access to GPIOA.

```c
RCC->AHB1ENR |= GPIOAEN;
```

Since the onboard LED is connected to PA5, clock access to GPIOA must be enabled before configuring or using the pin.

### iii.

Next we configure PA5 as an output pin.

```c
GPIOA->MODER |= (1U<<10);
GPIOA->MODER &= ~(1U<<11);
```

For Pin 5, bits 11:10 in the MODER register control the mode. Setting them to 01 configures PA5 as a General Purpose Output pin.

This LED will later be used as an indication that the DMA transfer has completed successfully.

### iv.

Next we initialize UART2.

```c
uart2_tx_init();
```

This configures PA2 as AF07, enables clock access to USART2, configures the baud rate, enables the transmitter and finally enables the USART2 module.

### v.

Next we initialize DMA1 Stream6.

```c
dma1_stream6_init((uint32_t)message,
                  (uint32_t)&USART2->DR,
                  31);
```

Three arguments are passed.

- The first argument is the address of the source buffer stored in memory.

```c
(uint32_t)message
```

- The second argument is the address of the USART2 Data Register.

```c
(uint32_t)&USART2->DR
```

This becomes the destination buffer.

- The third argument specifies the number of bytes to transfer.

```c
31
```

Inside the DMA initialization function, DMA1 Stream6 is configured for a Memory-to-Peripheral transfer. The source address, destination address and transfer length are configured. Memory increment mode is enabled so that DMA reads the next character after every transfer. Transfer Complete Interrupt is also enabled and finally DMA requests are enabled for USART2 transmission.

After Stream6 is enabled, DMA starts transferring data automatically.

The data flow becomes:

```text
Message in SRAM
↓
DMA reads character
↓
DMA writes character to USART2->DR
↓
USART2 transmits character
↓
Character appears on terminal
```

Unlike the UART TX Polling project, the CPU is not involved in transmitting each character. DMA handles the entire transfer.

### vi.

After initialization, the processor enters the infinite loop.

```c
while(1)
{

}
```

The CPU is now free while DMA performs the transfer in the background.

## DMA1_Stream6_IRQHandler()

### i.

After DMA transfers all 31 bytes, the transfer is completed.

At this point the DMA hardware automatically sets the TCIF6 (Transfer Complete Flag) bit in the HISR (High Interrupt Status Register).

<img width="900" height="400" alt="image" src="https://github.com/user-attachments/assets/8bcb09f7-ce95-4c6d-8203-43127e9883cb" />

Since we previously enabled the TCIE (Transfer Complete Interrupt Enable) bit and also enabled the DMA interrupt in the NVIC using:

```c
NVIC_EnableIRQ(DMA1_Stream6_IRQn);
```

an interrupt is generated and the processor automatically jumps to:

```c
void DMA1_Stream6_IRQHandler(void)
```

### ii.

Inside the ISR we first check whether the TCIF6 bit is set.

```c
if(DMA1->HISR & HISR_TCIF6)
```

If this condition is true, it means the DMA transfer has completed successfully.

### iii.

Next we clear the interrupt flag.

```c
DMA1->HIFCR |= HIFCR_CTCIF6;
```

Writing 1 to the CTCIF6 bit in the HIFCR register clears the corresponding TCIF6 flag in the HISR register.

<img width="900" height="400" alt="image" src="https://github.com/user-attachments/assets/fb934cfc-7def-4c70-95b9-315e70af906e" />

This prevents the interrupt from remaining pending and avoids the same interrupt from being serviced again.

### iv.

After clearing the flag, we execute:

```c
dma_callback();
```

## DMA Callback Function

### i.

Inside the callback function:

```c
GPIOA->ODR |= LED_PIN;
```

the PA5 bit in the Output Data Register is set.

This drives PA5 HIGH and turns ON the onboard LED.

The LED acts as a visual indication that the DMA transfer has completed successfully.
